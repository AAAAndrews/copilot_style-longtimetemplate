# 项目阶段交接记录

本文件用于多模块项目的阶段总结与交接。每完成一个模块，追加一个阶段记录。

## 记录格式

```markdown
### 阶段 N: 模块名（状态）
- 任务：Txxx-Tyyy（对应 task-list.json 中的任务范围）
- 版本：vX.Y
- 时间：YYYY-MM-DD ～ YYYY-MM-DD
- 摘要：该模块解决的问题与交付物
- 核心输出：
  - 文件/组件 1
  - 文件/组件 2
- 关键约束/限制：
  - 约束 1
  - 约束 2
- 依赖于：（若依赖前置模块，列在这里）
  - 阶段 N-1: xxx
- 下一步：下一个模块的初步计划
```

## 示例

### 阶段 1: 因子挖掘（完成 ✓）
- 任务：T001-T004
- 版本：v1.0
- 时间：2026-03-22 ～ 2026-04-05
- 摘要：构建因子库与回测框架，支持单因子与组合因子研究
- 核心输出：
  - `src/factor_mining/`: 因子计算模块
  - `src/backtest/`: 回测引擎
  - `data/factors.csv`: 因子输出库
- 关键约束/限制：
  - 仅支持日频数据
  - 单机运行，无分布式
- 下一步：交易接口需调用因子库生成信号

### 阶段 2: 交易接口（规划中）
- 任务：T101-T103
- 版本：v1.0 (计划)
- 时间：计划 2026-04-06 ～ 2026-04-20
- 摘要：构建交易信号接口，支持实时生成买卖点
- 核心输出：
  - `src/trading_interface/`: 交易信号生成模块
  - `src/order_executor/`: 订单执行逻辑
  - API 文档
- 关键约束/限制：
  - 暂未连接真实交易所
  - 模拟盘验证为主
- 依赖于：
  - 阶段 1: 因子挖掘
- 下一步：集成因子挖掘与交易接口，完整流程端到端验证

### 阶段 3: 风险控制（未开始）
- 任务：T201-T203
- 版本：v1.0 (计划)
- 摘要：增加风险管理，支持头寸管理、止损、风险预警
- 核心输出：
  - `src/risk_management/`: 风险管理模块
- 依赖于：
  - 阶段 1: 因子挖掘
  - 阶段 2: 交易接口
- 下一步：评估是否需要实时部署

## 如何使用本文件

1. **规划新模块**：在"规划中"或"未开始"状态下添加。
2. **启动新模块**：修改状态改为"进行中"，记录开始时间。
3. **完成一个模块**：
   - 修改状态改为"完成 ✓"
   - 记录结束时间
   - 更新"核心输出"中的实际文件/功能
   - 明确记录"下一步"，帮助下一个模块的开发者快速理解
4. **多人协作**：新成员可以快速读此文件，了解全局进度和模块依赖关系。
